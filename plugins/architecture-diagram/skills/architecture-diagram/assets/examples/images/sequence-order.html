Order Processing Flow

E-commerce order creation with inventory check and payment
Request Flow
Customer sends POST /orders through the API Gateway to Order Service. The gateway routes and validates the incoming request before forwarding it to the appropriate service handler.
Inventory & Payment
Order Service orchestrates a verification loop: checking stock availability with Inventory Service, then charging the payment via Payment Service. Both services confirm before proceeding.
Confirmation
After successful payment, items are reserved asynchronously. The order is confirmed with HTTP 201 and the confirmation propagates back through the gateway to the customer.
Message Types
4 synchronous calls, 1 asynchronous fire-and-forget (Reserve items), and 4 return messages. The loop fragment groups the stock check, payment charge, and their responses.
